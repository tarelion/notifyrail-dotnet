# NotifyRail README Hero

## Overview
README opening hero for NotifyRail, a reliable notification delivery backend.
The image should be wide, instantly readable, and less dense than the system
overview infographic.

## Learning Objectives
The viewer will understand:
1. NotifyRail is a reliable notification delivery backend.
2. The core pipeline moves from API to reports through a PostgreSQL queue and worker.
3. The project is validated by tests.

## Design Instructions

### Style Preferences
- Use `pop-laboratory`.
- Use technical precision, grid, coordinate markers, and marker highlights.
- Avoid Mermaid-like diagrams.

### Layout Preferences
- Use `bento-grid`.
- Use true 16:9 wide hero composition.
- Use minimal text.

### Other Requirements
- Do not show dates, versions, release numbers, serial numbers, fake barcodes, or fake metadata.
- Do not include endpoint paths.
- Keep all labels large and readable.
